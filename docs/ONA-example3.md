# RQ2: 집단 수준의 조절은 두 과제에서 CKC 단계와 어떻게 맞물리는가

## 연구 개요

Zabolotna et al.(2025)은 서로 다른 두 CSCL 과제—개념적 물리 포스터 제작 과제(SimSketch)와 실습 로봇 조립 과제(Simua)—에서 협력적 지식 구성(CKC)과 집단 수준의 학습 조절이 시간적으로 어떻게 맞물리는지를 Transmodal Ordered Network Analysis(T/ONA)로 분석하였다.

---

## T/ONA란 무엇인가

T/ONA는 학습자 간 상호작용 데이터에서 행동 코드들 사이의 **시간적 순서와 방향성**을 가진 연결망을 계산하고 시각화하는 분석 기법이다. 단순히 어떤 행동이 함께 나타났는지(공출현)만 보는 것이 아니라, **어떤 행동이 어떤 행동보다 먼저 나타나는 경향**이 있는지까지 포착한다.

### 네트워크 읽는 법

| 요소 | 의미 |
|------|------|
| **노드(점)** | 각 행동 코드 |
| **노드 크기** | 다른 코드들과의 연결 빈도 |
| **노드 내부 원** | 자기전이(self-transition) 빈도 — 같은 코드가 연속 반복된 정도 |
| **엣지 굵기** | 두 코드 간 연결 빈도 |
| **엣지 방향(화살표)** | 더 자주 먼저 나타나는 코드의 방향 |
| **노드 위치** | 다른 코드들과 맺는 관계 패턴의 총합 (수학적으로 결정됨) |
| **노드 간 거리** | 두 코드가 유사한 연결 패턴을 가지는 정도 |

### 축의 의미

- **x축 (Means Rotation)**: 두 집단(SimSketch vs. Simua) 간 차이를 최대화하는 방향 — 분산 14.9% 설명
- **y축 (SVD)**: 전체 데이터 내 집단 간 변산을 가장 많이 설명하는 방향 — 분산 23.7% 설명

> 두 축 모두 연구자가 임의로 긋는 것이 아니라 데이터 패턴을 수학적으로 추출한 결과다.

## 분석에 사용된 변인

### 코드 목록

#### CKC 코드 (상호 배타적 — 한 에피소드에 하나만 부여)

| 코드명 | 설명 |
|--------|------|
| `Share_Compare` | 정보 공유·비교 — 자신의 생각을 말하고 동료의 생각과 비교 |
| `Disagree` | 불일치·불협화음 탐색 — 반론 제기, 의견 충돌 |
| `Negotiate` | 공유 의미 협상·공동 구성 — 절충안 탐색, 함께 의미 만들기 |
| `Test_Modify` | 가설 검증·수정 — 제안된 이해를 확인하거나 수정 |

> `Agree`(합의 도달)는 빈도가 너무 낮아 분석에서 제외됨

#### 조절 코드 (비배타적 — CKC와 동시에 나타날 수 있음)

| 코드명 | 설명 |
|--------|------|
| `Emo_Mot` | 감정·동기 조절 — 부정적 감정을 긍정으로 전환, 집단 분위기 유지 |
| `Cogn` | 인지 조절 — 과제 이해 부족을 인식하고 전략을 조정 |

### 집단 정보

| 과제 | 대상 | 인원 | 집단 수 | 집단 크기 |
|------|------|------|---------|-----------|
| SimSketch | 8학년 | 34명 (여 78%) | 9개 | 3~4명 |
| Simua | 9학년 | 15명 (여 20%) | 5개 | 3명 |

### T/ONA 파라미터

| 파라미터 | 값 | 설명 |
|----------|-----|------|
| 분석 단위 | 의미 있는 에피소드 | 행동 시작 ~ 종료 또는 주제 이탈 시점 |
| Stanza window | 29.6ms | 코드 평균 지속 시간 — 이 안에 나타난 코드 간 연결 누적 |
| 연결 정의 | 동시 발생 또는 순차 발생 | 방향성 있는 연결(directed connection)로 누적 |

## 주요 결과 요약

### SimSketch — 조절이 협상을 선도하는 수렴적 구조

```
Cogn   ──→ Negotiate
Emo_Mot ──→ Negotiate
```

- 인지 조절 또는 감정·동기 조절이 먼저 작동한 후 협상으로 이어지는 패턴이 지배적
- Emo_Mot는 Share_Compare, Negotiate, Test_Modify와 균형 있게 연결되며 Negotiate 방향이 강함
- Cogn → Negotiate 방향이 명확하여 인지 조절이 공유 의미 형성을 능동적으로 이끔
- Emo_Mot 자기전이 존재 — 감정 조절이 반복적으로 정교화됨
- centroid가 x축 왼쪽(조절+Negotiate 쪽)에 위치

### Simua — 정보 공유와 감정 조절이 순환하는 분산적 구조

```
Emo_Mot ←→ Share_Compare (양방향 순환)
Negotiate ──→ Cogn (역전)
```

- 감정·동기 조절과 정보 공유가 방향 없이 서로를 촉발하며 반복
- Cogn은 Share_Compare, Disagree와 주로 연결 — 불일치 해소에 초점
- Negotiate → Cogn 방향: 협상 중 어려움이 생긴 후 인지 조절이 사후적으로 개입
- centroid가 x축 오른쪽(초기 CKC 코드 쪽)에 위치

### 두 과제의 핵심 대비

| 비교 항목 | SimSketch | Simua |
|-----------|-----------|-------|
| 인지 조절과 협상의 방향 | Cogn → Negotiate | Negotiate → Cogn |
| 감정 조절의 기능 | CKC를 향한 심리적 발판 | 협력 유지 자체가 목적 |
| Emo_Mot 주요 연결 대상 | Negotiate | Share_Compare (양방향) |
| 전체 구조 | 수렴적 (조절 → 협상) | 분산적 (공유 ↔ 감정 순환) |
| centroid x축 위치 | 왼쪽 (조절+Negotiate) | 오른쪽 (초기 CKC) |

### 통계적 유의성

- 두 과제의 x축 centroid 신뢰구간이 겹치지 않음 → 구조적으로 유의미하게 다름
- Mann-Whitney U 검정 결과 Share_Compare ↔ Disagree 간 전이 패턴만 통계적으로 유의
- 나머지 엣지는 방향 경향성은 존재하나 통계적 유의성은 없음

## 샘플 데이터 생성 가이드

### 데이터 구조

```
집단ID, 타임스텝, CKC코드, Emo_Mot(0/1), Cogn(0/1)
```

- CKC 코드는 타임스텝마다 하나만 부여 (상호 배타적)
- Emo_Mot, Cogn은 해당 타임스텝에 동시 발생 가능 (0 또는 1)

### SimSketch 시뮬레이션 규칙

| 시퀀스 패턴 | 확률 |
|-------------|------|
| Cogn → Negotiate | 높음 |
| Emo_Mot → Negotiate | 높음 |
| Emo_Mot → Test_Modify | 중간 |
| Share_Compare → Disagree | 낮음 |
| 에피소드 평균 지속 | 길게 (약 60~70초) |
| 집단당 CKC 빈도 | 낮게 (평균 10회) |

### Simua 시뮬레이션 규칙

| 시퀀스 패턴 | 확률 |
|-------------|------|
| Share_Compare → Emo_Mot | 높음 |
| Emo_Mot → Share_Compare | 높음 (양방향) |
| Negotiate → Cogn | 중간 |
| Share_Compare → Disagree | 높음 |
| 에피소드 평균 지속 | 짧게 (약 35~40초) |
| 집단당 CKC 빈도 | 높게 (평균 29회) |
