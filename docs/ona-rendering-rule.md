이 문서는 프론트엔드 시각화(D3.js 또는 Canvas)를 담당하는 에이전트가 ONA(Ordered Network Analysis) 네트워크 그래프를 화면에 렌더링할 때 준수해야 하는 시각적 규칙과 디자인 가이드라인입니다. ONA는 ENA와 유사하지만, 연결의 방향성을 삼각형과 쉐브론으로 표현한다는 고유한 특징을 가집니다.

## 1. 공간 좌표 및 위치 등록 (Co-registration)
*   **좌표 계산 루틴:** 노드들은 2차원 평면에 무작위로 배치되는 것이 아닙니다. 알고리즘은 **각 단위의 ONA 점수(Point)와 해당 네트워크 센트로이드 간의 거리를 최소화하는 '공동 등록(Co-registration)' 최적화 루틴**을 통해 노드를 공간상에 고정합니다.
*   **공간 의미:** 따라서 렌더링된 공간은 노드의 위치를 기준으로 해석되어야 합니다. 예를 들어, 특정 분석 단위(사용자)를 나타내는 점(Point)이 오른쪽 공간에 치우쳐 있다면, 해당 사용자는 공간의 오른쪽에 위치한 노드들 간의 연결을 더 빈번하게 발생시켰다는 뜻입니다.

## 2. 노드 (Nodes) 시각화 규칙
*   **반응 크기 (Node Size):** 특정 코드를 나타내는 노드의 전체 크기는 데이터 내에서 **해당 코드가 다른 코드에 대한 '반응(response)'으로서 발생한 횟수(상대적 빈도)에 비례**하여 커지도록 렌더링해야 합니다.
*   **자기 연결 (Self-connections):** 각 노드의 중앙에는 색칠된 원(inner circle)을 렌더링해야 합니다. 이 **내부 원의 색상과 채도, 크기는 해당 코드의 '자기 연결' 빈도를 의미**합니다. (예: 특정 정보에 반응하여 동일한 코드를 다시 반응으로 나타낸 경우).

## 3. 엣지 (Edges - Broadcast Model) 시각화 규칙
*   **브로드캐스트 모델 (Triangles):** ONA의 방향성 연결은 화살표나 단순한 선으로 그리지 않습니다. 방향성을 나타내기 위해 정보의 **출처(Ground) 측이 꼭짓점(apex)이 되고, 정보의 목적지이자 반응(Response) 측이 밑변(base)이 되는 '브로드캐스트(broadcast)' 형태의 두 겹친 삼각형 쌍**으로 렌더링해야 합니다.
*   **두께와 채도 (Thickness and Saturation):** 삼각형의 **두께와 채도는 두 코드 사이에서 발생한 반응(연결)의 강도와 상대적 빈도에 비례**하도록 렌더링합니다. 두꺼울수록 강한 연결을 의미합니다.

## 4. 쉐브론 (Chevrons - 방향성 기호) 규칙
*   **방향 표시:** 삼각형 모양의 엣지 위에는 공통 기반(Ground)에서 반응(Response)으로 이어지는 방향성을 직관적으로 보여주기 위해 **어두운 쉐브론(chevron, 꺾쇠) 기호**를 표시해야 합니다.
*   **양방향 연결 처리:** 동일한 두 노드 사이에 양방향(A->B, B->A) 연결이 모두 존재할 경우, 화면의 혼란을 방지하기 위해 **둘 중 연결 강도가 더 강한 쪽에만 쉐브론을 렌더링**합니다.
*   **동일 강도 처리:** 만약 양방향의 연결 강도가 완전히 동일하다면, 예외적으로 양쪽 방향 엣지 모두에 쉐브론을 표시합니다.

## 5. 센트로이드(Centroids) 및 신뢰구간 (Confidence Intervals)
*   **개별 점 (Points):** 분석 단위(개별 학습자)의 ONA 점수는 ONA 네트워크 공간상에 단일 점(Point)으로 렌더링됩니다. (이전 dENA 모델에서는 2개의 점이 필요했지만, ONA에서는 단일 고차원 벡터로 압축되어 1개의 점만 나타납니다).
*   **그룹 평균 (Means):** 집단의 평균 네트워크를 그릴 때, 집단 평균 점은 사각형(Square) 아이콘으로 렌더링하여 개별 점과 구분해야 합니다.
*   **신뢰구간 (Confidence Intervals):** 각 그룹의 평균 점 주변으로 차원별 95% 신뢰구간을 점선 박스(Box) 형태로 렌더링하여 통계적 분포를 보여줍니다.

## 6. 빼기 네트워크 (Subtracted Network) 시각화 규칙
*   **빼기 계산:** 두 집단을 시각적으로 비교하기 위해, **한 네트워크의 각 연결 가중치에서 다른 네트워크의 대응되는 가중치를 뺀 값**을 시각화하는 빼기 네트워크 모드를 지원해야 합니다.
*   **색상 대비 (Color Coding):** 빼기 네트워크의 엣지는 **두 네트워크 중 더 강한 연결을 가진 쪽의 색상**으로 렌더링됩니다 (예: 실험군이 강하면 파란색, 대조군이 강하면 빨간색). 
*   **가중치 스케일링 (Multiplier):** 연결의 차이가 미미할 경우 시각적 가독성을 높이기 위해 엣지 렌더링 시 **차이값에 배수(multiplier, 예: x4 또는 x5)를 곱하여 굵기를 재조정**할 수 있는 옵션을 제공해야 합니다.
