# ONA Calculation Logic (ona_calculation_logic.md)

이 문서는 ONA(Ordered Network Analysis) 웹 도구의 백엔드 API 및 데이터 처리 로직을 구현하는 에이전트를 위한 핵심 계산 파이프라인 지침입니다. ONA는 협력적 문제 해결이나 학습 과정에서 이벤트의 순서(Order)가 중요하게 작용하는 맥락을 모델링하기 위해 비대칭(방향성) 네트워크를 계산합니다.

## 1. 이동 스탄자 창(Moving Stanza Window)을 통한 연결 식별
*   **개념:** ONA 알고리즘은 현재 데이터 줄(반응, response)과 창 내의 이전 줄들(공통 기반, ground) 간에 형성된 연결을 식별하기 위해 이동 스탄자 창을 사용합니다.
*   **구현 지침:** 
    *   데이터를 스캔할 때 지정된 크기(예: 7)의 슬라이딩 윈도우를 적용합니다.
    *   창 내부에서 현재 발생한 이벤트(Response)와 그보다 앞서 발생한 이벤트(Ground)를 짝지어 방향성 있는 연결을 추출합니다.

## 2. 비대칭 인접 행렬(Asymmetric Adjacency Matrix) 및 벡터화
*   **개념:** ONA는 연결이 발생하는 인과적 순서를 고려하여 각 분석 단위(예: 개별 학습자)에 대해 비대칭 인접 행렬을 구성합니다. 
*   **구현 지침:**
    *   행렬 구성 시, 코드 A에서 코드 B로의 연결 빈도(A -> B)는 코드 B에서 코드 A로의 연결 빈도(B -> A)와 다르게 개별적으로 누적되어야 합니다.
    *   자신에서 자신으로 이어지는 '자기 연결(Self-connections, 예: A -> A)'의 빈도 또한 기록해야 합니다.
    *   단위별로 구성된 이 비대칭 인접 행렬을 단일 고차원 비대칭 인접 벡터(single high dimensional asymmetric adjacency vector)로 변환(Flatten)합니다.

## 3. 구면 정규화(Spherical Normalization)
*   **개념:** 분석 단위(학습자)마다 상호작용의 총량이나 활동 횟수(발화량 등)가 다를 수 있으므로, 이를 통제하기 위해 정규화 과정이 필수적입니다.
*   **구현 지침:**
    *   각 단위의 누적된 비대칭 인접 벡터를 해당 벡터의 전체 길이(Magnitude)로 나누어 구면 정규화를 수행합니다.
    *   이 과정을 거친 정규화 벡터의 각 값은 상대적인 코드 동시 발생 빈도를 나타내며, 0과 1 사이의 합리적인 수치(rational numbers)로 변환됩니다.

## 4. 차원 축소(Dimensional Reduction) 및 ONA 점수 계산
*   **개념:** 정규화된 고차원 벡터 데이터를 2차원 공간의 ONA 점수로 변환하여 시각화 및 통계 분석이 가능하게 만듭니다. ONA는 특이값 분해(SVD)와 평균 회전(Means Rotation, MR) 기법을 사용합니다.
*   **구현 지침:**
    *   **단일 집단 분석 (SVD):** 별도의 비교 그룹이 없을 경우, 데이터를 특이값 분해(SVD)하여 가장 큰 분산을 설명하는 상위 2개의 차원(SVD1, SVD2)을 X축과 Y축으로 사용합니다.
    *   **두 집단 비교 (Means Rotation):** '적극적 계획 집단'과 '소극적 계획 집단'처럼 두 개의 뚜렷한 이산 그룹을 비교할 때는 평균 회전(Means Rotation)을 기본값으로 적용해야 합니다. 이 방법은 두 그룹의 평균점이 공간의 X축(MR1)에 최대한 가깝게 배치되도록 하여 집단 간의 차이를 극대화하여 보여줍니다.
    *   차원 축소의 결과로 각 분석 단위는 2차원 공간의 단일 점(Point)인 ONA 점수를 갖게 됩니다. 

## 5. 공동 등록(Co-registration) 최적화
*   **개념:** 계산된 ONA 점수(분석 단위의 위치)와 네트워크 그래프의 노드 위치가 수학적으로 일치하도록 맞추는 과정입니다.
*   **구현 지침:** ONA 알고리즘은 각 단위의 ONA 점수(Point)와 해당 네트워크 센트로이드(Centroid) 간의 거리를 최소화하는 최적화 루틴을 통해 2차원 공간상에 노드들을 배치해야 합니다.
