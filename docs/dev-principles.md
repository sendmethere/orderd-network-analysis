초보 연구자가 ONA(Ordered Network Analysis)의 복잡한 수학적·이론적 개념을 쉽게 이해하고 연구에 바로 적용할 수 있도록 돕는 웹 도구 개발의 5대 대원칙은 다음과 같습니다.

**1. 직관적인 시각적 은유(Visual Metaphor)의 적극적 활용과 해설 제공**
초보 연구자들은 네트워크 그래프를 해석하는 데 어려움을 겪을 수 있으므로, ONA 특유의 시각적 요소가 의미하는 바를 UI/UX 상에서 직관적으로 해설해야 합니다. 
*   **노드와 반응성:** 노드의 전체 크기는 해당 코드가 다른 코드에 대한 '반응(response)'으로 나타난 상대적 빈도를 의미하며, 중앙의 색칠된 원은 자기 연결(self-connection)의 빈도를 나타냅니다. 
*   **엣지와 방향성:** 일반적인 화살표 대신 정보의 출처(ground)가 꼭짓점이 되고 반응(response)이 밑변이 되는 두 겹의 삼각형(브로드캐스트 모델)으로 엣지를 렌더링해야 합니다. 그 위에 그려진 쉐브론(꺾쇠) 기호는 정보의 인과적 흐름(출처 $\rightarrow$ 반응)을 명확히 보여줍니다.
*   초보자가 요소에 마우스를 올렸을 때(Hover), 이러한 맥락적 의미를 툴팁으로 즉시 설명해주어 해석의 장벽을 낮추어야 합니다.

**2. 대조를 통한 패턴 발견 지원: 빼기 네트워크(Subtracted Network)의 강조**
연구의 핵심은 종종 서로 다른 두 집단(예: 실험군과 대조군)의 차이를 규명하는 것입니다. 웹 도구는 두 집단의 평균 네트워크 차이를 시각화하는 **'빼기 네트워크' 기능을 핵심 기능으로 제공**해야 합니다. 
*   이를 위해 알고리즘은 평균 회전(Means Rotation)을 사용하여 두 집단 평균 사이의 거리를 극대화하는 공간을 구성해야 합니다.
*   빼기 네트워크 상에서 각 엣지는 두 그룹 중 더 강한 연결을 가진 쪽의 색상으로 표시되며, 두께는 그 차이의 크기를 나타냅니다. 이 대조적 시각화를 통해 초보 연구자는 집단 간 행동 순서의 질적인 차이를 쉽게 파악할 수 있습니다.

**3. 정량적 수치와 정성적 데이터의 연결: '해석적 루프 닫기(Closing the Interpretative Loop)' 구현**
초보 연구자가 알고리즘의 결과를 신뢰하고 깊이 있는 질적 분석을 수행하려면, 통계적 모델링 결과와 원본 데이터 사이를 오갈 수 있어야 합니다. 
*   연구자가 웹 도구에서 특정 엣지(연결선)를 클릭하면, 해당 연결을 생성하는 데 기여한 **원본 로그 데이터와 이동 스탄자 창(Moving Stanza Window)의 내역을 팝업이나 사이드 패널로 직접 확인할 수 있도록 구현**해야 합니다.
*   이러한 기능은 정량적 분석 결과가 정성적 원본 데이터를 공정하게 대변하는지 확인하는 '해석적 루프 닫기'를 가능하게 합니다.

**4. 복잡한 계산 파이프라인의 자동화 및 추상화**
ONA는 데이터를 분석 단위별 비대칭 인접 행렬(asymmetric adjacency matrix)로 축적하고, 이를 구면 정규화(spherical normalization)한 뒤 차원 축소(SVD 또는 MR)를 거치고, 공동 등록(Co-registration)을 통해 노드를 공간에 배치하는 복잡한 수학적 과정을 거칩니다. 
*   개발 시 이러한 **백엔드 파이프라인은 철저히 자동화하여 초보자가 복잡한 수학을 몰라도 도구를 사용할 수 있게 추상화**해야 합니다.
*   대신, 사용자는 '이동 스탄자 창(Stanza Window)의 크기(예: 7)'와 같은 직관적인 파라미터만 UI 슬라이더로 손쉽게 조정하여 모델이 어떻게 변하는지 확인할 수 있어야 합니다.

**5. 친숙한 도메인과 샘플 데이터를 통한 실습 환경(Sandbox) 제공**
초보 연구자는 자신의 데이터를 올리기 전, 잘 정제된 샘플 데이터로 도구의 작동 방식을 익혀야 합니다. 
*   교육적 맥락이 뚜렷한 **'퓨쳐월드(Futureworld)'와 같은 게임 기반 학습의 로그 데이터를 기본 샘플로 제공**합니다.
*   이 데이터를 바탕으로 '적극적 계획 집단'과 '소극적 계획 집단'이 문제 해결 과정에서 타일 교체(Tile Change), 읽기(Reading), 계획 도구(Planning Tool) 사용 등의 행동을 어떤 순서로 수행하는지 미리 정의된 변인으로 시연해 주어야 합니다. 이를 통해 사용자는 ONA가 시계열적 행동 패턴을 어떻게 포착하는지 구체적인 맥락 속에서 학습할 수 있습니다.
