# UI/UX Interactions & Contextual Explanations (ui_ux_interactions.md)

이 문서는 Next.js 기반의 ONA 웹 도구에서 UI/UX 컴포넌트(React)를 구성하는 에이전트가 참조할 지침입니다. 사용자가 직관적으로 ONA 그래프를 이해할 수 있도록 마우스 Hover, 툴팁(Tooltip), 그리고 사이드 패널 등을 통해 제공해야 할 맥락 정보와 시각적 요소 해설 텍스트를 구조화합니다.

## 1. 시각적 요소 해설 (Visual Elements Explanation)
그래프의 특정 요소에 마우스를 올렸을 때(Hover) 나타나야 할 툴팁 가이드라인입니다.

*   **노드 크기 (Node Size):**
    *   **해설 텍스트:** "노드의 크기는 이 행동이 이전 상황에 대한 '반응(Response)'으로서 얼마나 자주, 능동적으로 선택되었는지를 나타냅니다."
*   **자기 연결 (Self-connections - 내부 원):**
    *   **해설 텍스트:** "노드 중앙의 색칠된 원은 동일한 행동이 연이어 발생한 '자기 연결'의 빈도와 강도를 의미합니다."
*   **엣지 두께와 형태 (Edges - Triangles):**
    *   **해설 텍스트:** "두 노드를 연결하는 삼각형의 두께와 채도가 높을수록 두 행동 간의 연속적인 발생 빈도(연결 강도)가 강하다는 것을 의미합니다. 꼭짓점은 정보의 출처(Ground)를, 밑변은 반응(Response)을 나타냅니다."
*   **쉐브론 기호 (Chevrons - 방향성):**
    *   **해설 텍스트:** "삼각형 위의 꺾쇠(Chevron) 기호는 어떤 행동에서 어떤 행동으로 이어졌는지 인과적 흐름과 순서를 보여줍니다. 양방향 연결이 존재할 경우, 화면 혼란을 방지하기 위해 더 빈번하게 발생한(강도가 강한) 방향에만 표시됩니다."

## 2. 공간적 맥락 및 센트로이드 해설 (Spatial Context & Centroids)
사용자가 화면의 공간 배치나 분석 단위(점/사각형)를 클릭했을 때 제공할 정보입니다.

*   **노드의 공간 배치 (Co-registration):**
    *   **해설 텍스트:** "노드가 위치한 공간은 무작위가 아닙니다. 알고리즘(공동 등록)을 통해 노드와 사용자의 점 사이 거리가 최소화되도록 배치됩니다. 특정 노드들이 오른쪽이나 왼쪽에 모여 있다면, 해당 차원(가로/세로축)이 설명하는 주된 행동 패턴을 의미합니다."
*   **개별 점과 그룹 평균 (Points & Means):**
    *   **해설 텍스트:** "공간 상의 작은 점(Point)은 개별 학습자의 전반적인 행동 패턴 위치를 나타냅니다. 사각형(Square) 아이콘은 그룹 전체의 평균 위치를 나타내며, 점선 박스는 95% 신뢰구간을 의미하여 통계적인 분포를 보여줍니다."
    *   **해석 가이드:** "사용자의 점이 특정 노드들이 위치한 공간(예: 오른쪽)에 치우쳐 있다면, 해당 사용자가 그 노드들 사이의 연결을 훨씬 더 빈번하게 발생시켰다는 것을 뜻합니다."

## 3. 샘플 데이터(Futureworld) 도메인 맞춤형 해설 
'퓨쳐월드(Futureworld)' 게임 데이터를 기반으로 한 두 집단 비교 시나리오에서 패널(Panel)에 렌더링해야 할 도메인 특화 해설입니다.

*   **변인 설명 패널:**
    *   "이 데이터는 지속 가능 발전 교육용 게임에서 수집되었습니다. 자기 조절 학습을 위해 '계획 도구(PlanningTool)'와 '움직임 내역(MoveHistory)'을 사용하며, 문제 해결을 위해 '타일 교체(TileChange)', '읽기(Reading)', '다이어그램(Diagram)' 등의 학습 행동을 수행합니다."
*   **빼기 네트워크 (Subtracted Network) 해설 패널:**
    *   **해설 텍스트:** "빼기 네트워크 모드에서는 두 집단(적극적 계획 집단 vs 소극적 계획 집단) 간의 연결 가중치 차이를 대비되는 색상으로 시각화합니다. 한 집단에서 더 두드러지게 나타난 행동 흐름을 직관적으로 파악할 수 있습니다."
    *   **패턴 해석 가이드:** "계획 도구 사용 빈도가 높은 '적극적 계획 집단(파란색)'은 계획 도구 사용 후 읽기나 다이어그램 보기와 같은 학습 행동으로 이어지는 연결이 더 강하게 나타납니다. 반면 '소극적 계획 집단(빨간색)'은 문제 해결 행동인 타일 교체 중심의 단순 반복 패턴(예: 타일 교체 후 문제 종료 또는 재시작)이 더 자주 관찰됩니다."

## 4. UI 상호작용(Interaction) 동작 지침
Next.js 컴포넌트 개발 에이전트를 위한 추가 지시사항:
*   **하이라이팅 (Highlighting):** 사용자가 특정 노드에 Hover 했을 때, 해당 노드와 직접 연결된 삼각형 엣지 및 쉐브론만 불투명도(Opacity)를 100%로 유지하고, 나머지 그래프 요소는 흐리게(Dim) 처리하여 연결 관계에 집중할 수 있도록 구현하십시오.
*   **차이값 배수 스케일링 (Multiplier Slider):** 빼기 네트워크의 차이가 미미할 경우 시각적 가독성을 높일 수 있도록, 차이값에 곱해지는 배수(Multiplier)를 조정할 수 있는 슬라이더 UI를 제공하십시오.
